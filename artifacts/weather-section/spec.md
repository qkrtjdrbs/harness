# 현재 날씨 섹션

## 개요
홈페이지에 사용자가 나라를 선택해 해당 나라의 현재 날씨를 확인할 수 있는 섹션을 추가한다.

## 범위

### 포함
- 홈페이지(`app/page.tsx`)에 날씨 섹션 추가
- 전 세계 국가를 검색으로 찾아 선택할 수 있는 나라 선택 UI
- 페이지 첫 진입 시 한국 날씨를 기본으로 표시
- 선택한 나라의 현재 온도(°C), 날씨 상태, 습도, 바람 정보 표시
- 국가 변경 시 로딩 인디케이터 표시 후 새 날씨 데이터로 교체
- 날씨 데이터를 가져오지 못했을 때 단일 공용 에러 메시지 표시

### 제외
- 온도 단위 전환(화씨) — 섭씨로 고정하기로 확정됨
- 도시 단위 선택 — 나라 단위 대표 날씨만 제공하기로 확정됨
- 자동 주기 갱신 — 페이지 로딩·국가 변경 시에만 갱신하기로 확정됨
- 시간별/일별 예보 — 요구사항에 "현재 날씨"만 언급되어 예보는 범위 밖
- 로그인/개인화(즐겨찾는 나라 저장 등) — 요구사항에 언급 없음

## 시나리오

### 1. 페이지 진입 시 기본 날씨 표시
- **Given** — 사용자가 홈페이지에 처음 접속한다
- **When** — 페이지가 로딩을 마친다
- **Then** — 날씨 섹션에 한국의 현재 온도, 날씨 상태, 습도, 바람 정보가 표시된다

성공 기준:
- [ ] 페이지 접속 → 나라 선택 UI에 "대한민국"(또는 한국)이 선택된 상태로 표시된다
- [ ] 로딩 완료 → 온도(°C 단위), 날씨 상태 텍스트, 습도(%), 바람 정보가 모두 화면에 표시된다

### 2. 다른 나라 검색 후 선택
- **Given** — 한국 날씨가 표시된 화면
- **When** — 나라 선택 UI에서 "일본"을 검색해 선택한다
- **Then** — 로딩 인디케이터가 잠시 나타난 뒤 일본의 현재 날씨 정보로 화면이 바뀐다

성공 기준:
- [ ] "일본" 검색 → 검색 결과 목록에 "일본"이 나타난다
- [ ] "일본" 선택 → 온도·날씨 상태·습도·바람 정보가 일본 날씨 값으로 교체된다

### 3. 검색어에 맞는 나라가 없음
- **Given** — 나라 선택 UI가 열려 있다
- **When** — 존재하지 않는 국가명을 검색어로 입력한다
- **Then** — 검색 결과가 없다는 안내가 표시되고, 이전에 선택된 나라의 날씨는 그대로 유지된다

성공 기준:
- [ ] 존재하지 않는 국가명 입력 → "검색 결과 없음" 문구가 나타난다
- [ ] 결과 없음 상태 → 기존에 표시되던 날씨 정보는 변경되지 않는다

### 4. 날씨 데이터 로딩 중
- **Given** — 사용자가 나라를 선택해 새 날씨 데이터를 요청한 직후
- **When** — 데이터를 가져오는 중이다
- **Then** — 로딩 인디케이터만 표시되고, 완료되면 결과 또는 에러로 전환된다

성공 기준:
- [ ] 나라 선택 직후 → 로딩 인디케이터가 표시된다
- [ ] 로딩 완료 → 인디케이터가 사라지고 날씨 정보(또는 에러 메시지)로 대체된다

### 5. 날씨 데이터 조회 실패
- **Given** — 사용자가 나라를 선택했다
- **When** — 날씨 데이터를 가져오는 데 실패한다
- **Then** — 단일 공용 에러 메시지가 날씨 섹션에 표시된다

성공 기준:
- [ ] 조회 실패 → 날씨 섹션에 공용 에러 메시지가 표시된다
- [ ] 에러 표시 상태에서도 나라 선택 UI는 계속 사용할 수 있어 다른 나라를 다시 선택할 수 있다
