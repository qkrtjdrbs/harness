# 웹 페이지 → Markdown 변환 서비스

## 개요
사용자가 웹 페이지 URL을 붙여넣으면 본문을 Markdown으로 추출해 화면에 보여주고, 그 결과를 복사·다운로드하거나 프롬프트와 함께 ChatGPT/Claude로 바로 넘길 수 있게 한다.

## 범위

### 포함
- URL 입력, 지우기(입력 필드만 초기화)
- defuddle 기반 본문 → Markdown 추출
- 페이지 제목·저자 헤더 + 렌더링된 Markdown 결과 표시
- 결과 복사하기 / .md 파일 다운로드 (항상 순수 Markdown, 프롬프트 미포함)
- 프롬프트 프리셋(요약해줘 / 한국어로 번역해줘 / 쉽게 설명해줘) 또는 직접 입력 선택
- "ChatGPT로 열기" / "Claude로 열기": 프롬프트(선택 시) + Markdown을 클립보드에 복사한 뒤 해당 서비스 새 탭 열기
- 다크모드 토글, 선택값은 재방문 시에도 유지

### 제외
- 로그인/계정 기능 — 도구가 세션 간 개인화 데이터를 다루지 않으므로 불필요
- 변환 이력 저장/조회 — 요구사항에 언급 없음, 매 방문 1회성 변환으로 한정
- 직접 입력 프롬프트 저장 — 요구사항에 "1회용(저장하지 않음)"으로 명시됨
- 페이지 로딩/추출 취소 — 대부분 몇 초 내 종료되는 짧은 작업으로 간주 (사용자 확인)
- ChatGPT/Claude 페이지에 텍스트 자동 붙여넣기 — 두 서비스 모두 URL로 텍스트를 주입하는 공식 방법이 없어 클립보드 복사 + 새 탭 열기로 대체 (사용자 확인)

## 시나리오

### 1. URL 변환 성공 (제목·저자 모두 있음)
- **Given** — 빈 URL 입력 필드가 있는 초기 화면
- **When** — 제목과 저자 정보를 가진 페이지의 URL을 붙여넣고 변환을 실행한다
- **Then** — 페이지 제목과 저자가 헤더에 표시되고, 그 아래 렌더링된 Markdown 본문이 나타난다

성공 기준:
- [ ] 유효한 아티클 URL 입력 → 제목 텍스트와 저자 텍스트가 헤더에 표시된다
- [ ] 변환 완료 → 원문 본문에 대응하는 렌더링된 Markdown이 결과 영역에 표시된다

### 2. 제목/저자 정보가 없는 페이지 변환
- **Given** — 저자 메타데이터가 없는 페이지의 URL
- **When** — 해당 URL로 변환을 실행한다
- **Then** — 저자 필드는 화면에 나타나지 않고, 나머지 헤더·본문은 정상 표시된다 (제목이 없는 경우도 동일하게 해당 필드만 생략)

성공 기준:
- [ ] 저자 정보 없는 페이지 → 저자 필드가 화면에 존재하지 않는다 (빈 텍스트가 아니라 요소 자체가 없음)
- [ ] 제목 정보 없는 페이지 → 제목 필드가 화면에 존재하지 않는다

### 3. 변환 실패
- **Given** — 접근 불가/봇 차단/추출 실패 등으로 변환할 수 없는 URL
- **When** — 해당 URL로 변환을 실행한다
- **Then** — 단일 공용 에러 메시지가 표시되고, 사용자는 URL을 수정해 다시 시도할 수 있다

성공 기준:
- [ ] 변환 불가능한 URL 입력 → 공용 에러 메시지가 표시된다
- [ ] 에러 메시지 표시 후 → URL 입력 필드는 그대로 남아 있어 즉시 재시도 가능하다

### 4. 변환 중 로딩 상태
- **Given** — URL을 입력하고 변환을 실행한 직후
- **When** — 페이지 로딩과 Markdown 추출이 진행 중이다
- **Then** — 로딩 인디케이터만 표시되고, 완료되면 결과 또는 에러로 전환된다

성공 기준:
- [ ] 변환 실행 직후 → 로딩 인디케이터가 표시된다
- [ ] 변환 완료 → 로딩 인디케이터가 사라지고 결과(또는 에러 메시지)로 대체된다

### 5. 지우기 버튼
- **Given** — 텍스트가 입력된 URL 필드 (결과가 이미 표시된 상태 포함)
- **When** — 지우기 버튼을 클릭한다
- **Then** — URL 입력 필드만 빈 값으로 초기화되고, 기존 결과·프롬프트 선택 상태는 그대로 유지된다

성공 기준:
- [ ] 결과가 표시된 상태에서 지우기 클릭 → URL 입력 필드는 빈 문자열이 된다
- [ ] 지우기 클릭 → 기존에 표시되던 결과 영역은 변경되지 않고 그대로 남는다

### 6. 결과 복사하기
- **Given** — 변환이 완료되어 Markdown 결과가 표시된 화면 (프롬프트 프리셋 선택 여부 무관)
- **When** — "복사하기"를 클릭한다
- **Then** — 프롬프트를 포함하지 않은 순수 Markdown이 클립보드에 복사되고, 복사 완료를 알리는 표시가 나타난다

성공 기준:
- [ ] 프롬프트가 선택된 상태에서 복사하기 클릭 → 클립보드 내용은 프롬프트 없이 Markdown 본문만 포함한다
- [ ] 복사하기 클릭 → 복사 완료를 알리는 시각적 피드백이 나타난다

### 7. .md 파일 다운로드
- **Given** — 변환이 완료되어 Markdown 결과가 표시된 화면
- **When** — "다운로드"를 클릭한다
- **Then** — 프롬프트를 포함하지 않은 순수 Markdown이 `.md` 파일로 다운로드된다

성공 기준:
- [ ] 다운로드 클릭 → `.md` 확장자 파일이 다운로드된다
- [ ] 다운로드된 파일 내용 → 프롬프트 텍스트 없이 결과 화면의 Markdown과 동일하다

### 8. 프롬프트 프리셋 선택 후 ChatGPT/Claude로 열기
- **Given** — 변환이 완료된 화면에서 프롬프트 프리셋(요약해줘 / 한국어로 번역해줘 / 쉽게 설명해줘) 중 하나를 선택한 상태
- **When** — "ChatGPT로 열기" 또는 "Claude로 열기"를 클릭한다
- **Then** — 선택한 프리셋 문구가 앞에 붙은 Markdown 전체가 클립보드에 복사되고, 해당 서비스(chatgpt.com 또는 claude.ai) 새 탭이 열린다

성공 기준:
- [ ] "요약해줘" 선택 후 ChatGPT로 열기 → 클립보드 내용이 "요약해줘"로 시작하고 이어서 Markdown 본문을 포함한다
- [ ] ChatGPT로 열기 클릭 → chatgpt.com 새 탭이 열린다
- [ ] Claude로 열기 클릭 → claude.ai 새 탭이 열린다

### 9. 직접 입력 프롬프트로 열기
- **Given** — 프리셋 옆 "직접 입력"을 선택해 텍스트 인풋에 문구를 입력한 상태
- **When** — "ChatGPT로 열기" 또는 "Claude로 열기"를 클릭한다
- **Then** — 입력한 문구가 앞에 붙은 Markdown이 클립보드에 복사되고 새 탭이 열린다

성공 기준:
- [ ] 직접 입력 "핵심만 3줄로 요약해" 입력 후 열기 → 클립보드 내용이 해당 문구로 시작한다
- [ ] 새로고침 후 "직접 입력" 텍스트 인풋 → 이전에 입력한 문구가 남아있지 않고 비어 있다

### 10. 프롬프트 없이 열기
- **Given** — 프리셋/직접 입력 모두 선택하지 않은 상태
- **When** — "ChatGPT로 열기"를 클릭한다
- **Then** — 프롬프트 없이 Markdown 본문만 클립보드에 복사되고 새 탭이 열린다

성공 기준:
- [ ] 프롬프트 미선택 상태에서 열기 클릭 → 클립보드 내용이 Markdown 본문으로 시작한다 (프롬프트 문구 없음)

### 11. 다크모드 토글
- **Given** — 라이트 모드로 화면을 보고 있는 사용자
- **When** — 다크모드 토글을 켠다
- **Then** — 화면이 다크 테마로 즉시 전환되고, 페이지를 새로고침하거나 다시 방문해도 다크 테마가 유지된다

성공 기준:
- [ ] 다크모드 토글 켜기 → 화면 배색이 다크 테마로 바뀐다
- [ ] 다크모드 켠 뒤 새로고침 → 다크 테마가 그대로 유지된다

## 불변 규칙

- 복사하기/.md 다운로드 결과에는 어떤 경우에도 프롬프트 문구가 포함되지 않는다 — 프롬프트는 "ChatGPT·Claude로 열기" 액션에만 적용된다.
- 로딩 중에는 취소 동작이 존재하지 않는다 — 로딩 인디케이터 표시 후 결과 또는 에러로만 전환된다.
